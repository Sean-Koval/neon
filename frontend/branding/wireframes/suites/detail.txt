================================================================================
SUITE DETAIL — app/suites/[id]/page.tsx
================================================================================
Suite definition detail: metadata, test cases, run history, score trends.
Route: /suites/[id]

DESIGN NOTES:
  1. The detail page is the operational hub for a single suite. It answers:
     "What does this suite test, how has it been performing, should I run it?"
  2. Test cases are the core content — each case is expandable to show full
     input JSON, expected outputs, scorer config, and tags.
  3. Run history provides temporal context: is this suite trending up or down?
     The score trend chart and run history table answer this at a glance.
  4. "Run Suite" is the primary CTA — always visible in the header.
  5. Edit/Delete are secondary actions. Delete requires confirmation.
  6. The page layout is a single scrollable column (no tabs) since all sections
     are typically relevant together.
  7. EvalCase fields:
     - input: Record<string, unknown> (JSON, the data sent to the agent)
     - expected_tools: string[] (unordered tools that should be called)
     - expected_tool_sequence: string[] (tools in exact order)
     - expected_output_contains: string[] (strings in output)
     - expected_output_pattern: string (regex for output)
     - scorers: ScorerType[] (per-case override of suite defaults)
     - scorer_config: Record<string, unknown> (per-scorer config overrides)
     - min_score: number (0-1)
     - tags: string[]
     - timeout_seconds: number (1-3600)
  8. Suite execution config: parallel (bool), stop_on_failure (bool).
     These are secondary information, not prominent.

DATA: trpc.suites.get(suiteId), trpc.evals.listRuns({ suiteId, limit: 10 })

DATA SOURCES:
  - trpc.suites.get(suiteId) → EvalSuite (includes cases[] array)
  - trpc.evals.listRuns({ suiteId, limit: 10 }) → EvalRun[] (run history)
  - Summary stats derived from most recent completed run:
    - Last Pass Rate: summary.passed / summary.total_cases
    - Avg Score: summary.avg_score
    - Total Runs: count from listRuns response
  - Score trend chart data: map runs → [{ runNumber, avgScore, passRate, date }]
  - Delete: trpc.suites.delete({ suiteId })
  - Run Suite: opens existing StartEvalRunDialog component
  - Edit Suite: links to /suites/[id]/edit (existing wizard in edit mode)
  - Agent link: /agents/[suite.agent_id]

BEHAVIORAL NOTES:
  1. All cases collapsed by default. Clicking a case header toggles it.
     Only one case expanded at a time? No — allow multiple open simultaneously.
  2. The score trend chart uses the same recharts line chart component pattern
     as other trend charts in the app. Click data points to navigate.
  3. Run history table shows the last 10 runs. "View All Runs" links to
     the eval-runs page pre-filtered by suite_id.
  4. Delete is protected by a confirmation dialog. After deletion,
     redirect to /suites with a success toast.
  5. "Run Suite" always opens StartEvalRunDialog (existing component),
     never navigates directly. The dialog handles version selection,
     trigger type, and execution.
  6. If a suite has cases with custom scorer overrides (different from
     suite defaults), the case card shows the overridden scorers. If
     identical to suite defaults, shows "Suite defaults" to reduce noise.
  7. The min_score threshold line on the chart uses the suite's
     default_min_score value.
  8. Cache: staleTime 5 minutes for suite detail, 2 minutes for run history
     (runs are more dynamic).
  9. Back button uses router.back() if referrer is /suites, otherwise
     links directly to /suites.
